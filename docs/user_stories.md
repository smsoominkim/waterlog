# 사용자 스토리 (User Stories)

이 문서는 물 마시기 습관 형성 서비스의 핵심 기능을 사용자 입장에서 서술한 요구사항 명세서입니다.

## 1. 인증 (Authentication)

### 1.1 로그인
- **사용자**로서,
- 나는 **서비스에 로그인**하여 내 기록을 안전하게 관리하고 싶다.
- **인수 기준 (Acceptance Criteria):**
    - 이메일/비밀번호 또는 소셜 로그인(예: Google)을 지원해야 한다.
    - 로그인 성공 시 메인 대시보드로 이동한다.
    - 로그인 실패 시 적절한 에러 메시지를 보여준다.

### 1.2 로그아웃
- **사용자**로서,
- 나는 **내 계정에서 로그아웃**하여 개인정보를 보호하고 싶다.
- **인수 기준:**
    - 로그아웃 버튼을 누르면 즉시 로그아웃 처리된다.
    - 로그아웃 후에는 로그인 페이지로 리다이렉트된다.

---

## 2. 물 마신 기록 관리 (Water Intake Management)

### 2.1 물 마신 기록 남기기
- **사용자**로서,
- 나는 **물을 마실 때마다 그 양과 시간을 기록**하여 하루 섭취량을 추적하고 싶다.
- **인수 기준:**
    - 물의 양(ml)을 입력할 수 있어야 한다. (예: 200ml, 500ml 등 프리셋 제공 가능)
    - 기록 시점의 날짜와 시간이 자동으로 저장되어야 한다. (수동 수정 가능 옵션 있으면 좋음)
    - '기록하기' 버튼을 누르면 즉시 리스트에 반영되어야 한다.

### 2.2 기록 조회하기
- **사용자**로서,
- 나는 **오늘 혹은 과거에 마신 물 기록을 조회**하여 내 습관을 확인하고 싶다.
- **인수 기준:**
    - 날짜별로 섭취 기록 리스트를 볼 수 있어야 한다.
    - 일별 총 섭취량이 계산되어 표시되어야 한다.
    - (선택 사항) 주간/월간 통계 그래프가 있으면 좋다.

---

## 3. AI 기반 리포트 (AI-based Report)

### 3.1 AI 습관 분석 리포트
- **사용자**로서,
- 나는 **내 물 마시기 패턴을 분석한 AI 리포트**를 받아보고 더 나은 습관을 위한 조언을 얻고 싶다.
- **인수 기준:**
    - 쌓인 데이터를 바탕으로 AI(Gemini API 활용)가 분석한 리포트를 생성해야 한다.
    - 리포트에는 섭취 패턴 분석, 칭찬, 개선할 점 등이 포함되어야 한다.
    - 리포트는 주기적(예: 매일 밤 혹은 주간) 혹은 사용자가 요청할 때 생성된다.
